Provider decision matrix (five processes x 25 samples): single-axis consumed repeat is a confirmed baseline CPU gap (+13.77%, +0.480 ms), Accelerate parity (+6.79%), Metal gap (+37.84%, +0.646 ms), and CUDA 59.97% faster. The same view lowering is 99.99% faster to construct on CPU, two-axis consume is 73% faster on CPU/Accelerate, 39% faster on Metal, and 84% faster on CUDA. There is no implementation point at which add_axes can detect a future contiguous call. Provider-eager lowering would violate the documented storage-sharing API and regress the larger winning set. README now documents the explicit eager-repeat alternative for callers that know immediate materialization is required.
